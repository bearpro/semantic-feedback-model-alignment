Опиши на C# модель, описывающую данную вещь.

Требования к модели:
- Верни только один C# файл.
- Не пиши пояснений вне кода.
- Модель должна отражать только сведения из исходного текста.
- Не добавляй знания извне.

Правила семантического именования:
1. Имя типа должно обозначать одну устойчивую сущность предметной области в единственном числе.
2. Имя свойства должно обозначать ровно один наблюдаемый атрибут, часть, роль или связь.
3. Не используй расплывчатые имена: Data, Info, Params, Details, Item, Value, Object, Thing, Settings, Config, Meta.
4. Для количественных величин явно отделяй величину от единицы измерения:
   - например WidthMm, VoltageV, CapacityWh, DurationMin.
5. Для булевых свойств используй префиксы Is/Has/Can/Requires только если это действительно булево условие.
6. Для идентификаторов используй суффикс Id.
7. Для коллекций используй имя во множественном числе только если это действительно коллекция.
8. Если в тексте есть несколько альтернативных терминов, выбери один основной термин и используй его везде последовательно.
9. Не создавай два разных имени для одной и той же сущности.
10. Если свойство описывает часть объекта, имя должно отражать именно часть, а не общий контейнер.
11. Если свойство описывает отношение между сущностями, имя должно отражать отношение, а не случайный контекст употребления.
12. Избегай сокращений, кроме общеизвестных и технически однозначных.

Формат:
- Используй namespace GeneratedModel;
- Предпочитай record/class и enum;
- Не добавляй методов, кроме тех, которые критически нужны для структуры модели.
